Техническое описание проекта "50 кубов красного"

Обзор проекта:
Игра "50 кубов красного" разработана для платформы Яндекс Игр с использованием Unity. Это головоломка, в которой игрок управляет красными и синими кубиками, чтобы направить берёзовый кубик к целевому отверстию через 10 уникальных уровней. Проект был создан за несколько дней с акцентом на быстрое прототипирование и интеграцию с инфраструктурой Яндекс Игр.

Техническая реализация:
- Версия Unity: Unity 2022.3.37f1
- Язык программирования: C#
- Система сохранений: Интеграция с SDK Яндекс Игр для облачного сохранения прогресса.
- Система рекламы: Реализована интеграция рекламы Яндекс Игр для показа межуровневых объявлений, запускаемых при завершении уровня или определённых игровых событиях. Настроена частота показа рекламы для баланса между монетизацией и пользовательским опытом.
- Игровая механика:
  - Движение кубиков: Красные кубики ограничены вертикальным движением, синие — горизонтальным, реализовано через манипуляцию Rigidbody или Transform с обработкой ввода (мышь/клавиатура).
  - Дизайн уровней: 10 уникальных уровней, созданных вручную с использованием системы сцен Unity. Каждый уровень включает красные зоны (убийственные триггеры), коричневые зоны, кнопки и неподвижные белые кубики-препятствия.
  - Обработка столкновений: Использована физическая система Unity (2D или 3D, в зависимости от реализации) для взаимодействия кубиков и проверки границ.
- Интерфейс: Простой интерфейс для выбора уровней и игровой обратной связи, создан с использованием системы UI Unity.

Проблемы, с которыми столкнулся:
- Интеграция SDK Яндекса: Проблемы с настройкой SDK из-за недостаточно подробной документации. Решено путём изучения форумов сообщества и тестирования.
- Оптимизация производительности: Для работы на слабых устройствах оптимизирована логика движения кубиков и минимизированы физические расчёты.
- Дизайн уровней: Сложность в балансировке уровней. Проведены итерации для обеспечения постепенного роста сложности при сохранении доступности игры.
- Интеграция рекламы: Настройка частоты показа рекламы для предотвращения раздражения игроков потребовала нескольких тестовых сборок.

Уроки, вынесенные из проекта:
- Быстрая разработка в Unity позволяет оперативно создавать прототипы, но требует чёткого планирования, чтобы избежать расширения объёма задач.
- SDK Яндекс Игр имеет специфические требования к облачным сохранениям и рекламе, что требует тщательного тестирования.
- Создание интуитивно понятных, но сложных головоломок критично для удержания игроков в коротких играх.

Возможные улучшения:
- Добавление новых уровней для повышения реиграбельности.
- Введение новых типов кубиков или механик (например, вращающиеся кубики).
- Улучшение визуальной обратной связи для взаимодействия с кубиками и завершения уровней.
